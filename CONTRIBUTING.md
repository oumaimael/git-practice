To keep things organized, please follow these branch naming conventions:
- `feat/feature-name` : For new features.
- `fix/bug-name` : For bug fixes.
- `docs/document-name` : For documentation updates.
- `style/style-name` : For styling/design changes.
- `refactor/refactor-name` : For code refactoring.

## ðŸ’¬ Commit Conventions
We use **Simple Conventional Commits**. Your commit messages should be descriptive and follow this format:
- `feat: add modulo operator`
- `fix: correct display overflow`
- `docs: update readme screenshot`
- `style: change button border radius`

## ðŸš€ Workflow: How to Contribute

1.  **Fork the Project**: Create a copy of this repository on your own GitHub account.
2.  **Clone the Repository**:
    ```bash
    git clone https://github.com/YOUR_USERNAME/funky-calculator.git
    cd funky-calculator
    ```
3.  **Create a Branch**: Use the naming rules mentioned above.
    ```bash
    git checkout -b feat/your-funky-feature
    ```
4.  **Implement Changes**: Ensure your code remains minimalist and funky!
5.  **Verify Integrity**: 
    - Run `npm install` to ensure all dependencies are correct.
    - Run `npm run dev` and test your changes in the browser.
    - Check for any console errors.
6.  **Create a Pull Request (PR)**:
    - Push your branch to your fork.
    - Open a PR against the `main` branch of the original repository.
    - Fill out the PR template with details about your changes.
